## 2. Interpret

All 14 tools were interpreted directly from their Zod schema declarations in
index.ts. Key observations:

- Required vs optional parameters: declared explicitly in each Zod schema.
  No ambiguity; required fields use z.string() or z.array(); optional fields
  use .optional() or .default().

- Path restriction: all tools route through validatePath() before any
  filesystem operation. Paths outside allowed directories are rejected.
  Allowed directories are set at startup via CLI args and may be overridden
  at runtime by MCP Roots protocol.

- Destructive tools: write_file, edit_file, move_file. write_file overwrites
  without confirmation. edit_file requires exact text match.

- read_file is a documented deprecated alias for read_text_file. Both are
  registered as separate tools in the server manifest.

- No duplicated functionality except the deprecated alias pair.

No ambiguity was encountered. All fields were populated from the source.
